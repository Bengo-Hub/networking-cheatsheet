# Contributing to Networking Cheatsheet

Thank you for your interest in contributing to the Networking Cheatsheet repository!

## How to Contribute

### Reporting Issues

If you find any errors, typos, or have suggestions for improvement:

1. Check if the issue already exists in the Issues section
2. Create a new issue with a clear title and description
3. Provide as much context as possible (sections affected, screenshots if applicable)

### Contributing Content

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-contribution`)
3. Make your changes
4. Ensure your changes follow the existing documentation style
5. Commit your changes (`git commit -m 'Add amazing contribution'`)
6. Push to the branch (`git push origin feature/amazing-contribution`)
7. Open a Pull Request

## Guidelines

### Documentation Standards

- Use clear, concise language
- Provide practical examples where applicable
- Include code blocks with syntax highlighting when showing configurations
- Maintain consistency with existing formatting
- Verify all configuration examples are accurate

### Markdown Formatting

- Use proper heading hierarchy
- Include tables of contents for longer documents
- Use code blocks for all configuration examples
- Add comments to explain complex configurations

### Content Areas

We welcome contributions in:

- Additional router/switch configuration examples
- IPv6 deployment strategies and best practices
- Networking fundamentals and educational content
- Lab setup guides and troubleshooting tips
- Security best practices
- Network design principles

## Code of Conduct

- Be respectful and constructive in all interactions
- Focus on technical accuracy and educational value
- Help maintain a welcoming environment for all contributors

Thank you for helping improve this resource!
